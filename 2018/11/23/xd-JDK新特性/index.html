Jdk 8

Lanmbda

概述

  • Lambda 是一个 匿名函数，我们可以把 Lambda表达式理解为是 一段可以传递的代码（将代码像数据一样进行传递）
  • Lambda需要函数式接口的支持

并行流/串行流

并行流

  • 把一个内容分成多个数据块，并用不同的线程分别处理每个数据块的流

Fork/Join框架

  • 采用 “工作窃取”模式（work-stealing）
    • 当执行新的任务时它可以将其拆分分成更小的任务执行，并将小任务加到线程队列中，然后再从一个随机线程的队列中偷一个并把它放在自己的队列中
  • 与一般线程池实现的比较
    • 在一般的线程池中,如果一个线程正在执行的任务由于某些原因无法继续运行,那么该线程会处于等待状态
    • 在fork/join框架实现中,如果某个子问题由于等待另外一个子问题的完成而无法继续运行.那么处理该子问题的线程会主动寻找其他尚未运行的子问题来执行

函数式接口

  • 接口中只有一个抽象方法，即为函数式接口
  • @FunctionalInterface
    • 标明是函数式接口

内置函数式接口

语法

  • 左右遇一括号省
    • 左侧参数为一省略左侧小括号
    • 右侧语句为一省略return和大括号
  • 左侧推断类型省

操作符->

  • 箭头操作符或Lambda操作符
  • 左侧
    • 指定了 Lambda 表达式需要的所有参数
    • (参数列表)
  • 右侧
    • 指定了 Lambda 体，即 Lambda 表达式要执行的功能
    • {lambda体}

格式

  1. () -> System.out.println("hello lambda")
    • 无参
  2. (args) -> System.out.println(args)
    • 一个参数
  3. args -> System.out.println(args)
    • 一个参数可以省略小括号()

  4. .

    • 两个参数，且又返回值
  5. (x,y) -> x+y;
    • lambda体只有一句时，return和{}可以省略

  6. .

    • 参数的类型可以省略，因为可由编译器推断得出，称为“类型推断”

类型推断

  • Lambda 表达式中无需指定类型，程序依然可以编译，这是因为 javac 根据程序的上下文，在后台推断出了参数的类型。
  • Lambda 表达式的类型依赖于上下文环境，是由编译器推断出来的。这就是所谓的“类型推断”

方法引用

  • 当要传递给Lambda体的操作，已经有实现的方法了，可以使用方法引用

前提

  • Lambda体方法引用的参数列表和返回值类型必须与函数式接口中抽象方法的参数和返回值保持一致

格式

  1. 对象::实例方法
  2. 类名::静态方法
  3. 类名::实例方法
    • 当lambda第一个参数是实例方法的调用者，且第二个参数是实例方法的参数（或无参数）
    • 如(x, y) -> x.equals(y)
    • String::equals

构造引用

  • 构造器参数列表与函数式接口中抽象方法的参数列表一致

格式

  • 类名::new

数组引用

格式

  • Type[]::new

Stream API

概述

  • java.util.stream .*
  • 是数据渠道，用于操作数据源（集合、数组等）所生成的元素序列
  1. Stream 自己不会存储元素
  2. Stream 不会改变源对象。相反，他们会返回一个持有结果的新Stream
  3. Stream 操作是延迟执行的。这意味着他们会等到需要结果的时候才执行
    • 即中间操作不会进行操作，终止操作时，一次性执行全部内容，即“惰性求值”

使用

创建Stream

Collection
  • default Stream<E> stream()
    • 返回一个顺序流
  • default Stream<E> parallelStream()
    • 返回一个并行流
Arrays
  • static <T> Stream<T> stream(T[] array)
Stream
  • static<T> Stream<T> of(T... values)
  • static<T> Stream<T> iterate(final T seed, final UnaryOperator<T> f)
  • static<T> Stream<T> generate(Supplier<T> s)

Optional

  • java.util.Optional 是一个容器类，代表一个值存在或不存在，
  • 原来用 null 表示一个值不存在，现在 Optional 可以更好的表达这个概念。并且可以避免空指针异常

API

  • static of(T t)
    • 创建Optional
  • static empty()
    • 创建一个空的Optional
  • static ofNullable(T t)
    • 若t不为null，创建Optional，否则创建空的Optional
  • isPresent()
    • 是否包含值
  • orElse(T t)
    • 如果调用对象包含值，返回该值，否则返回t
  • orElse(Supplier s)
    • 如果调用对象包含值，返回该值，否则返回s获得的值
  • map(Function f)
    • 如果有值对其处理，并返回处理后的Optional，否则返回Optional.empty()
  • flatMap(Function mapper)
    • 类似map，要求返回值必须为Optional
