集合 & 数组

区别

  1. .
    • 数组既可以存储基本数据类型（存储值），又可以存储引用数据类型（存储地址值）。
    • 集合只能存储引用数据类型，存储基本数据类型时会自动装箱为基本类型包装类。
  2. .
    • 数组长度是固定的,不能自动增长
    • 集合的长度的是可变的,可以根据元素的增加而增长

选择使用

  • 如果元素个数是固定的推荐用数组
  • 如果元素个数不是固定的推荐用集合

数据结构

数组 & 链表

  • 数组
    • 查询快修改也快
    • 增删慢
  • 链表
    • 查询慢修改也慢
      • 链表的查询是先判断索引在前半段还是后半段，然后一个一个查询过去。所以速度很慢。
    • 增删快

栈 & 队列

  • 栈
    • 先进后出
  • 队列
    • 先进先出

泛型

概述

  • 提高安全性(将运行期的错误转换到编译期)
  • 省去强转的麻烦
  • <>中放的必须是引用数据类型
  • 前后的泛型必须一致,或者后面的泛型可以省略不写(1.7的新特性菱形泛型)

泛型类

  • public class 类名<泛型类型1,…>

泛型接口

  • public interface 接口名<泛型类型1,…>
    • 实现类
      1. public 类名 implements 接口<具体类型>
        • 实现时直接指定类型，推荐
      2. public 类名<泛型> implements 接口<泛型>

泛型方法

  • public <泛型类型> 返回类型 方法名(泛型类型 变量名)
    • 泛型定义在修饰符后返回值前
      • 静态方法不能访问类上定义的泛型。只能自己定义泛型.
      • 非静态方法最好使用类上定义的泛型.

泛型通配符

  • <?>
    • 任意类型，如果没有明确，那么就是Object以及任意的Java类了
  • ? extends E
    • 向下限定，E及其子类（通常 存）
  • ? super E
    • 向上限定，E及其父类（通常 取）

集合

  • 单列集合底层依赖的是双列集合(安)
    • 双列变单列 简单
    • 单列变双列 难

interface Collection

  • 增加
    • boolean add(E e)
    • boolean addAll(Collection c)
  • 删除
    • boolean remove(E e)
    • boolean removeAll(Collection c)
    • void clear()
  • 判断
    • boolean contains(E e)
    • boolean containsAll(Collection c)
    • boolean isEmpty()
  • 获取
    • int size()
    • Iterator iterator()
  • other
    • boolean retainAll(Collection c)
      • 取交集
      • 调用的集合改变就返回true
    • Object[] toArray()
      • 集合→数组
    • T[] toArray(T[] t)
      • 传入一个指定类型的数组，并转为指定类型的数组
      • 传入的数组长度如何定义？
        • 如果长度小于集合的size，那么该方法会创建一个同类型并和集合相同size的数组。
        • 如果长度大于集合的size，那么该方法直接使用传入的数组，存储集合的元素，其它位置默认为null
        • 所以传入的长度最好为集合的size

interface Iterator

概述

  • Collection容器进行元素抽取的公共接口.
  • 只定义了取出元素的方式。具体实现依赖于具体的容器。
  • 具体的容器以内部类实现Iterator接口通过iterator方法返回对象。

方法

  • boolean hasNext()
  • E next()
  • void remove()

迭代器原理

  • 迭代器是对集合进行遍历,而每一个集合内部的存储结构都是不同的,所以每一个集合存和取都是不一样,那么就需要在每一个类中定义hasNext()和next()方法,这样做是可以的,但是会让整个集合体系过于臃肿,迭代器是将这样的方法向上抽取出接口,然后在每个类的内部,定义自己迭代方式。
  • 好处
    1. 规定了整个集合体系的遍历方式都是hasNext()和next()方法,
    2. 代码有底层内部实现,使用者不用管怎么实现的,会用即可
  • 迭代器源码解析
    • iterator()方法
    • 返回值类型是new Itr(),说明Itr这个类实现Iterator接口
    • 查找Itr这个内部类,发现重写了Iterator中的所有抽象方法

interface ListIterator

概述

  • 使用迭代器迭代时，如果对集合进行添加，会发生并发修改异常ConcurrentModificationException。
  • 使用ListIterator中的方法可以在迭代list集合时进行添加。

方法

  • void add(E e)
  • boolean hasNext()
  • E next()
  • boolean hasPrevious()
  • E previous()

interface Set

特点

  • 存取无序(存入和取出的顺序不一致)
  • 元素唯一(元素不可以重复)

HashSet

唯一

  • 元素要存入HashSet必须覆盖hashCode()和equals()方法
  • 哈希表判断元素是否相同
    1. 判断两个元素的哈希值是否相同
      • 使用的是对象的hashCode()方法
      • 如果哈希值不同，说明元素不相同。不用再判断equals了
    2. 如果哈希值相同，则判断内容是否相同
      • 使用equals()方法

LinkedHashSet

特点

  • 存取有序
  • 元素唯一

TreeSet

特点

  • 存取无序
  • 元素唯一
  • 元素排序

唯一

  • 保证元素唯一是根据比较方法来判断的。

排序

  1. 存入的元素具有比较性
    • 元素实现Comparable接口
      • 覆写compareTo()方法
  2. 集合具有比较性
    • 创建一个比较器类实现Comparator接口
    • 创建集合时传入比较器类。
      • 如果传入比较器，则会优先使用比较器，而不使用元素本身的比较性。
      • 这样我们可以利用自定义的比较器(元素比较时如果=0，返回一个非0的数)，就可以添加相同的元素。

      • 可以通过返回null判断是否包含指定key
    • int size()
  • 迭代
    • Set<k> keySet()
    • Set<Map.Entry<k,v>> entrySet()
      • Map.Entry是内部类接口的形式。
    • Collection<E> values()

TreeMap

HashMap

LinkedHashMap

HashTable

Properties

特点

  • Properties 类表示了一个持久的属性集。
  • Properties 可保存在流中或从流中加载。
  • 属性列表中的键和值都是字符串。
  • 通常用于操作以键值对形式存在的配置文件。

方法

  • Set<String> stringPropertyNames()

  • Object setProperty(String key, String value)
    • 设置键值，调用的是Hashtable的put方法
  • Enumeration<?> propertyNames()
    • 获取键的枚举
  • void list(PringtStream out)
    • 将集合的数据输出到指定的输出流
      • 如prop.list(System.out)即输出到控制台
    • 常用于调试
  • void list(PrintWriter out)
