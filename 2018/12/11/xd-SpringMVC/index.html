配置前端控制器：DispatcherServlet

  • 默认加载
    • WEB-INF/<servlet-name>标签的值-servlet.xml
    • 上例中即默认加载WEB-INF/springMVC-servlet.xml
  • <url-pattern>
    • /*（不建议）
      • 拦截所有，包含.jsp，包含(.js .png .css…..)
    • /（建议）
      • 拦截所有，不包含.jsp，包含(.js .png .css…..)
      • 多用在前台
    • *.action/do
      • 拦截指定扩展名
      • 多用在后台

注解

@RequestMapping

作用

定义在类上

  • 第一级的访问目录
  • @RequestMapping("/user")
    • 是为了确定哪个controller的方法
    • 请求路径为/user/xxx

定义在方法上

  • 第二级的访问目录

属性

  • String[] value
    • 与path通用，指定请求url
  • String[] path
  • RequestMethod[] method
    • 确定请求方式，不写表示所有请求方式都有用
    • RequestMethod.POST/RequestMethod.GET
  • String[] params
    • 用于指定限制请求参数的条件，支持简单的表达式。要求请求参数的key和value必须和配置的一样。
    • params={"username"}
      • 表示请求参数必须有username
    • params={"age=11"}
      • 表示请求参数必须有age=11
    • params={"money!100"}
      • 表示请求参数必须有money且值不为100
  • String[] headers
    • 发送的请求中必须包含的请求头
    • headers{"Accept"}
      • 表示请求头中必须有Accept

@RequestParam

作用

  • 把请求中的指定名称的参数传递给控制器中的形参赋值

属性

  • value
    • 请求参数中的名称
  • required
    • 请求参数中是否必须提供此参数
    • 默认true，必须提供

@RequestBody

作用

  • 用于获取请求体的内容（注意：get方法不可以）

属性

  • required
    • 是否必须有请求体，默认值是true

@PathVariable

作用

  • 拥有绑定url中的占位符的
    • /delete/{id}
      • {id}就是占位符

属性

  • value
    • 指定url中的占位符名称

@RequestHeader

作用

  • 获取指定请求头的值

属性

  • value
    • 请求头的名称

@CookieValue

作用

  • 用于获取指定cookie的名称的值

属性

  • value
    • cookie的名称

@ModelAttribute

作用

  • 方法上
    • 表示当前方法会在控制器方法执行前先执行
  • 参数上
    • 获取指定的数据给参数赋值

应用场景

  • 当提交表单数据不是完整的实体数据时，保证没有提交的字段使用数据库原来的数据

@SessionAttributes

作用

  • 用于多次执行控制器方法间的参数共享

属性

  • value
    • 指定存入属性的名称

请求参数绑定

概述

  • Springmvc通过命令设计模式接受页面参数
  • springmvc是单例的没有成员变量，把需要传递的参数放入方法中，请求这个方法时，会自动封装参数

默认参数绑定

  • Request
  • Response
  • Session
  • Model
  • ModelMap

基本类型 & String

  1. 请求参数名与方法形参一致
  2. 不一致
    • @RequestParam(value="id",required=false,defaultValue="1")
      • @RequestParam
        • 用于处理简单类型的绑定
      • value
        • 绑定指定名称的请求参数
      • required
        • true
          • 默认，表示请求中一定要有相应的参数，否则报错
      • defaultValue
        • 表示请求中如果没有同名参数时的默认值

POJO

  • 方法形参名称没什么用
  • 方法形参的POJO类型中的字段与页面的name属性相同

POJO包装类

  • 类似POJO类，多了一步导航
  • 对于包装类中的POJO，表单name使用对象导航图

异常处理思路

    • 有异常处理器，则交给异常处理器
    • 无异常处理器，则什么都不做直接响应到页面

  1. 自定义异常类
    • 用于设置异常信息

  2. 自定义异常处理器

    • 实现HandlerExceptionResoler

  3. 在springmvc.xml中配置异常处理器bean

拦截器

拦截器 & 过滤器

  1. 过滤器是Servlet规范的一部分，任何框架都可以使用过滤器技术。
  2. 拦截器是SpringMVC框架独有的。
  3. 过滤器配置了/*，可以拦截任何资源。
  4. 拦截器只会对Controller中的方法进行拦截

自定义拦截器

  1. 实现HandlerInterceptor
  2. 在springmvc.xml中配置拦截器类

HandlerInterceptor

  • preHandler()
    • controller方法执行前执行
    • 可以使用request或者response跳转到指定的页面
    • return true
      • 放行，执行下一个拦截器，如果没有拦截器，执行controller中的方法
    • return false
      • 不放行，不会执行controller中的方法
  • postHandler()
    • controller方法执行后，在JSP视图执行前，执行
    • 可以使用request或者response跳转到指定的页面
    • 如果指定了跳转的页面，那么controller方法跳转的页面将不会显示
  • afterCompletion
    • JSP视图执行后，执行
    • request或者response不能再跳转页面了

url模板映射

  • url模板映射可以实现restfull软件架构

restfull

  • 请求地址没有?参数,扩展名等等
  • 请求更简洁，更安全，更方便于搜索引擎收录

实现restfull

请求参数

  • 将请求参数使用{}加入到请求地址中
    • @RequestMapping("update/{id}")
  • 通过注解@PathVariable表明传递参数
    • @PathVariable Integer id
  • 请求url不再使用?
    • /user/update?user.id.do（原）
    • /user/update/${user.id}.do（restfull）

扩展名

  • 通过约定修改web.xml的servlet拦截地址
  • <url-pattern>/xxx/*</url-pattern>

JSON

  • <mvc:annotation-driven/>
    • 默认自动创建注解处理器映射器，注解处理器适配器。提供json格式支持

  • @ResponseBody & @RequestBody

    • @RequestBody接受前台json数据，把json数据自动封装javaBean
    • @ResponseBody把后台pojo转换json对象，返回到页面
    • 这两个注解依赖Jackson的jar包

多视图

  • xml格式支持需要的jar包spring-oxm.jar

  • 配置springmvc支持多视图

  • 通过以上配置还要在需要使用的POJO类上配置注解@XmlRootElement

    @XmlRootElement
public class User {}
  • 然后请求地址后缀名为
    • .json则返回json视图
    • .xml则返回xml视图

页面缓存

  • 使用oscache
  • 原理
    • 缓存数据结构：map
      • key：存储浏览器访问url
      • Value：缓存页面数据
  • 存储范围
    • 缓存默认存储在application域当中
  • 修改范围
    • <oscache:cache scope="session"></oscache:cache>
  • 固定缓存key
    • <oscache:cache key=""></oscache:cache>
  • 每隔固定时间(如3秒)同步一次
    • <oscache:cache time="3"></oscache:cache>
  • 缓存持久化
    • oscache.properties放在classpath下

Springmvc的freemarker支持

SpringMVc与Struts2的区别

    • sm入口是servlet
    • s2是filter
    • sm是基于方法开发（一个url对应一个方法），可以是单例也可以是多例（推荐单例）
    • s2是基于类开发，只能为多例
      • 单例是以方法为单位
      • 多例是以类为单位
    • sm的参数是绑定在方法形参上
    • s2的参数绑定在成员变量上
    • sm通过参数解析器将request请求内容解析，绑定到方法形参后，将数据和视图封装成ModelAndView对象，最后将ModelAndView中的模型数据通过request域传输到页面
    • s2采用值栈存储请求和响应数据，通过OGNL存取数据
