ECMAScript

概述

  • ECMAScript实际上是一种脚本在语法和语义上的标准。
  • ECMAScript是JavaScript的规格
  • JavaScript是ECMAScript的一种实现

实现

  • JavaScript
  • JScript
  • ActionScript

说明

  • 三个都是ECMAScript。但是在操作浏览器对象等方面又有各自独特的方法，这些都是各自语言的扩展。
  • 类似于ASCII码表，以后的码表都包含这个。这三个脚本语言都包含ECMAScript而又有各自的东西(安)

DHTML

  • 不是一门语言，是多项技术综合体的简称
  • html
    • 负责提供标签，进行数据的封装。便于对数据的操作。
    • 用标签封装数据。
  • css
    • 负责提供样式。
    • 对数据定义样式。
  • dom
    • 负责将标签文档解析，封装对象，便于操作。
    • 解析文档封装对象。
  • javascript
    • 负责提供程序设计语言，对页面的对象进行逻辑操作。
    • 页面的行为定义，就是页面的动态效果。
  • DHTML+XMLhttpRequest=AJAX

JavaScript

概述

  • 基于对象和事件驱动的解释性脚本语言。
    • 基于对象
      • JS本身就有一些现成的对象可供程序员使用，也可自定义对象。
    • 事件驱动
      • JS代码写好后，需要外界触发后，方可运行
      • 例如：单击事件，定时执行。。。
    • 解释性
      • 每次运行JS代码时，得需要将原代码一行一行的解释执行
      • 相对编译型语言（例如：Java、C++）执行速度相对较慢
    • 基于浏览器的动态交互网页技术
      • JS嵌入到HTML中，可以不需要服务器支持，直接由浏览器解释执行
      • JS嵌入到JSP或Servlet中，必需要服务器支持，直接由浏览器解释执行

特点

  • 交互性
  • 安全性
    • js不能访问本地磁盘
  • 跨平台

基于对象&面向对象(网)

  • 基于对象
    • 先有一个具体的对象，然后在这个对象的基础上创建新的对象
    • 一个工程师建了一栋房子，然后其它的工程师按照这个房子的样子去建造其它的房子
  • 面向对象
    • 先有一个抽象的对象描述，然后以此为蓝本构建具体对象
    • 一个工程师再图纸上设计出一栋房子的样子，然后其它工程师按照这个图纸的设计去建造房子

引入方式

  1. 内嵌脚本
    • <input type="button" value="button" onclick="alert('xxx')"/>
  2. 内部脚本
    • 将js代码封装到<script>标签中
    • <script type="text/javascript">js代码<script>
      • type可以省略 默认。
      • <script>标签可以出现在html页面的任意位置，但实际操作中，一般会写在<head>标签中。
      • 在不影响html功能的前提下越晚加载越好
  3. 外部脚本
    • 将js代码封装成js文件，使用时通过<script>标签的src属性导入
    • <script src="xx.js></script>"
      • 外部的js文件中不能包含<script>标签。
      • 在导入外部js文件的<script>标签中不能在写js代码(标签体为空)。

  • DOM level 3

解析

dom解析

  • w3c标准解析
  • 将标记型文档解析成DOM树，并将树中的内容都封装成节点对象。
    • DOM树
      • 按照标签的层次关系体现出标签的所属，形成的一个树状结构。
    • 节点(Node)
      • DOM树中的标签，属性及文本。
      • 节点也成为对象。
      • 标签通常也称为页面中的元素。
  • 好处
    • 可以对节点进行任意操作。
  • 弊端
    • 需要将整个标记型文档加载进内存，如果文档过大，浪费内存空间。

SAX解析

  • 民间方式。
  • 基于事件驱动的解析。
  • 依序读入文件并产生相应事件，可以处理任意大小的文件。
  • 只能读取，不能修改。不能往回读。
