Git简介

  • 分布式版本管理

git配置的增删改查

查

  • git config --list
    • git config -l简写

增

  • git config --global 键 '值'

删

  • git config --global --unset 键

改

  • git config --global --replace-all 键 '值'

创建仓库

  • git init
    • 将一个目录变成Git管理的仓库
    • 目录下会有一个隐藏的.git文件

版本库概念

  • 仓库包含
    • 版本库
      • .git
    • 工作区
      • 除了.git文件夹外的其它

  • git add
    • 把工作区的文件添加到版本库的暂存区

  • git commit
    • 把暂存区的文件提交到版本库的指定分支

添加 & 提交

  • git add
    • 把文件添加到版本库的暂存区
    • 注意文件编码
    • 可以多次add不同的文件
  • git commit [-m "message"]
    • 把文件提交到版本库的分支
    • -m
      • 提交的说明

历史版本

  • git status
    • 查看仓库状态
  • git diff
    • 比较工作区和暂存区文件的区别
  • git diff --cached
    • 比较暂存区和分支文件的区别
  • git diff HEAD
    • 比较工作区和分支文件的区别
  • git log [--pretty=oneline]
    • 查看历史记录
    • --pretty=oneline
      • 整理日志信息（在一行..）

版本回退

版本说明

  • HEAD
    • 表示当前版本
  • HEAD^
    • 上一个版本
  • HEAD^^
    • 上上个版本
  • HEAD-100
    • 上100个版本

命令

  • git reset --hard HEAD或commit id
  • git reflog
    • 记录着每一次命令（有命令id）
    • 回退后，可以用此命令查询commit id

撤销修改

工作区的文件修改了，尚未add

  • git checkout -- <file>
    • 撤销工作区文件的修改
    • 我的理解就是从版本库分支中检出了一份覆盖了。

工作区的文件修改了，已经add，尚未commit

  • git reset HEAD <file>
    • 将文件从暂存区回退到工作区
  • git checkout -- <file>

删除文件

  • rm <file>
    • 删除本地的（工作区的）
  • git checkout -- <file>
    • 不想删了，可以再检出回来
  • git rm <file>
  • git commit -m
    • 经过上面两步就把版本库中的也删除了

远程仓库

HTTPS

  • 不需要私匙和公匙，推送和克隆时需要邮箱和密码

SSH

  • Secure Shell
    • 安全外壳协议
  • 使用私匙和公匙推送和克隆

为什么GitHub需要SSH Key呢？

  • 因为GitHub需要识别出你推送的提交确实是你推送的，而不是别人冒充的，而Git支持SSH协议，所以，GitHub只要知道了你的公钥，就可以确认只有你自己才能推送

配置SSH KEY

  1. 创建SSH Key
    • 先看本地用户目录下是否有.ssh
      • id_rsa
        • 私钥，不能泄露出去
      • id_rsa.pub
        • 公钥，可以放心地告诉任何人
    • ssh-keygen -t rsa -C "youremail@example.com"
      • 如果没有，用此命令创建
  2. 登陆GitHub，打开“Account settings”，“SSH Keys”页面
    • Add SSH Key
      • Title
        • 任意
      • Key
        • id_rsa.pub文件的内容
        • 可以添加多个

从本地推送到远程库

  1. 在GitHub创建一个库
  2. 与本地仓库关联（在需要关联的仓库下执行命令）
    • git remote add origin git@github.com:awcnoanswer/test.git
      • 远程库默认名origin（别名）
  3. 推送本地内容到远程库
    • git push -u origin master
      • 第一次推送master分支时，
      • -u
        • Git不但会把本地的master分支所有内容推送的远程新的master分支，还会把本地的master分支和远程的master分支关联起来，
    • git push origin master 最新修改
      • 在以后的推送或者拉取时就可以简化命令

从远程库克隆到本地

  • git clone git@github.com:awcnoanswer/test.git
    • 会克隆到执行命令的目录

分支管理

概述

  • master
    • 主分支
  • dev
    • 分支
  • HEAD
    • 当前版本
  • 默认只有一条主分支，然后创建分支，想要切换分支，只要把HEAD指向想要切换的分支即可。
  • 分支合并到主分支
    • master指向分支即可。

创建&合并分支

  • git branch dev
    • 创建分支dev（自定义）
  • git checkout dev
    • 切换到分支dev
  • git checkout -b dev
    • git checkout命令加上-b参数表示创建并切换，相当于上两条
  • git branch
    • 查看当前分支
    • 会列出分支列表，当前分支前有*号
  • git merge dev
    • 把分支dev合并到当前分支
  • git branch -d dev
    • 删除分支dev

解决冲突

  1. git status
    • 查看冲突信息
  2. 手动修改冲突文件
  3. add commit
  • git log --graph
    • 查看分支合并图

分支管理策略

分支合并

  • git merge dev
    • 默认使用fast forward
      • 快进模式下，删除分支后，会丢掉分支信息
      • 合并就看不出来曾经做过合并。
    • --no-ff
      • 用普通模式合并，合并后的历史有分支，能看出来曾经做过合并

分支管理原则

  • master
    • 主分支应该是非常稳定的，也就是仅用来发布新版本，平时不能在上面干活
  • dev
    • 分支是不稳定的，到某个时候，比如1.0版本发布时，再把dev分支合并到master上，在master分支发布1.0版本，然后继续在分支干活准备2.0版本。
    • 平时在分支上干活

Bug分支

  • 正在分支进行工作，临时需要修复一个bug，然而分支工作没完成不能提交，
    1. 先保存当前分支工作空间
    2. 创建临时bug分支
    3. 修复完合并并删除bug分支
    4. 恢复分支工作空间
  • 注意?

命令

  • git stash
    • 保存工作空间
  • git stash list
    • 查看保存列表
  • git stash apply
    • 恢复工作空间，但不删除stash内容
  • git stash pop
    • 恢复工作空间，同时删除stash内容

feature分支

  • 附加功能分支，要添加新功能，重新开启一个分支
  • git branch -D <name>
    • 强制删除还没有合并的分支

多人协作

  • git remote
    • 查看远程仓库信息
  • git remote -v
    • 查看远程仓库详细信息
  • git checkout -b branch-name origin/branch-name
    • 将远程仓库origin中的分支branch-name检出到本地
  • git push origin branch-name
    • 将分支推送到远程仓库
    • 如果推送失败，先git pull更新，解决冲突后，再提交
  • gti pull
    • 抓取最新的提交
    • 如果失败提示no tracking information说明本地分支与远程分支的链接关系没有创建
      • git branch --set-upstream branch-name origin/branch-name

标签管理

创建标签

  • git tag <name>
    • 新建一个标签，默认为HEAD，也可以指定一个commit id
  • git tag -a <tagname> -m "blablabla..."
    • 可以创建时指定标签信息
  • git tag -s <tagname> -m "blablabla..."
    • 用PGP签名标签
    • 签名采用PGP签名，因此，必须首先安装gpg（GnuPG）
  • git tag
    • 查看所有标签，默认安装字母排序
  • git show <tagname>
    • 查看标签信息

操作标签

  • git push origin <tagname>
    • 推送一个本地标签到远程仓库
  • git push origin --tags
    • 推送全部未推送过的本地标签
  • git tag -d <tagname>
    • 删除一个本地标签；
  • git push origin :refs/tags/<tagname>
    • 删除一个远程标签。

自定义git

忽略文件

  • 在工作区创建.gitignore，然后把要忽略的文件名填进去.
    • .gitignore文件可以在线查询
  • 忽略文件的原则是：
    1. 忽略操作系统自动生成的文件，比如缩略图等；
    2. 忽略编译生成的中间文件、可执行文件等，也就是如果一个文件是通过另一个文件自动生成的，那自动生成的文件就没必要放进版本库，比如Java编译产生的.class文件；
    3. 忽略你自己的带有敏感信息的配置文件，比如存放口令的配置文件
