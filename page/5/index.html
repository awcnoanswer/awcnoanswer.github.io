类加载

概述

  • 当程序要使用某个类时，如果该类还未被加载到内存中，则系统会通过加载，连接，初始化三步来实现对这个类进行初始化。

加载

  • 就是指将class文件读入内存，并为之创建一个Class对象。任何类被使用时系统都会建立一个Class对象。

连接

  • 验证 是否有正确的内部结构，并和其他类协调一致
  • 准备 负责为类的静态成员分配内存，并设置默认初始化值
  • 解析 将类的二进制数据中的符号引用替换为直接引用

初始化

  • 实例对象初始化过程

时机

  • 创建类的实例
  • 访问类的静态变量，或者为静态变量赋值
  • 调用类的静态方法
  • 使用反射方式来强制创建某个类或接口对应的java.lang.Class对象
  • 初始化某个类的子类
  • 直接使用java.exe命令来运行某个主类

类加载器

概述

  • 负责将.class文件(可能在本地，也可能在内存)加载到内存中，并为之生成对应的Class对象。

分类&作用

Bootstrap ClassLoader

  • 根类加载器
  • 也被称为引导类加载器，负责Java核心类的加载
  • 比如System,String等。在JDK中JRE的lib目录下rt.jar文件中

Extension ClassLoader

  • 扩展类加载器
  • 负责JRE的扩展目录中jar包的加载。
  • 在JDK中JRE的lib目录下ext目录

Sysetm ClassLoader

  • 系统类加载器(AppClassLoader)
  • 负责在JVM启动时加载来自java命令的class文件，以及classpath环境变量所指定的jar包和类路径

父类委托机制

  • 保证字节码只被加载一次，避免类重复加载。
  • 要加载一个类
  • Sysetm ClassLoader首先让Extension ClassLoader找
  • Extension ClassLoader让Bootstrap ClassLoader找
  • Bootstrap ClassLoader找不到，Extension ClassLoader才找
  • Extension ClassLoader找不到，Sysetm ClassLoader才找
  • Sysetm ClassLoader找不到，ClassNotFoundException
  • 安
    • 总之就是一级一级找上边加载，最后找到最顶层，然后上边一级一级往下轮着加载，不管哪一级加载到就停止，一直到最底层如果还没加载到，就报错。
    • 从哪一层开始找，最后到那一层结束，所以class文件不能乱放

java执行过程

  1. .java原文件 编译
  2. .class字节码文件
  3. 类加载器把.class字节码文件装载到内存运行环境中，生成Class字节码对象。
  • 由上述过程可知，根据Class字节码对象可以知道是哪个类加载器加载的
    • ClassLoader cl = Test.clsss.getClassloader()

反射

概述

  • JAVA反射机制是在运行状态中
    • 对于任意一个类，都能够知道这个类的所有属性和方法，
    • 对于任意一个对象，都能够调用它的任意一个方法和属性。
  • 简言之：
    • 动态获取类的信息以及动态调用对象的方法的功能称为java语言的反射机制。
  • 反射要先获取到字节码文件对象。

获取字节码文件对象

  1. 类对象.getClass()
  2. 类名.class
  3. Class.forName(String className)

    • 开发者实现自己的servlet然后通过配置文件配置。
    • tomcat加载配置文件，通过反射访问具体实现类的信息。

xd-网络编程

发表于 2018-11-23 | 更新于 2018-12-27 | 分类于 Java

概述

  • 就是用来实现网络互连的不同计算机上运行的程序间可以进行数据交换。

三要素

IP

  • 每个设备在网络中的唯一标识
  • 每台网络终端在网络中都有一个独立的地址，我们在网络中传输数据就是使用这个地址。
  • ipconfig
    • 查看本机
  • ping
    • 测试连接
  • 127.0.0.1
    • 本地回路地址
  • localhost
    • 主机名
  • 255.255.255.255
    • 广播地址
  • IPv4
    • 4个字节组成，4个0-255。大概42亿，30亿都在北美，亚洲4亿。2011年初已经用尽。
  • IPv6
    • 8组，每组4个16进制数。

端口

  • 每个程序在设备上的唯一标识
  • 每个网络程序都需要绑定一个端口号，传输数据的时候除了确定发到哪台机器上，还要明确发到哪个程序。
  • 端口号范围从0-65535
  • 编写网络应用就需要绑定一个端口号，尽量使用1024以上的，1024以下的基本上都被系统程序占用了。
  • 常用端口
    • mysql: 3306
    • oracle: 1521
    • web: 80
    • tomcat: 8080
    • QQ: 4000

  • 创建DatagramSocket, 随机端口号,也可指定.

  • 使用DatagramSocket接收DatagramPacket

    • int port = packet.getPort()
      • 获取发送端口
    • String s = new String(packet.getData(), 0, packet.getLength())
      • 获取发送数据
  • 关闭DatagramSocket
    • receiveSocket.close()

子网掩码

概述

  • 随着互联网的发展，越来越多的网络产生，有的网络多则几百台主机，少则区区几台，这样就浪费了大量的IP地址，所以要划分子网。为了有序地区分和管理不同的子网，就需要使用子网掩码。
  • 子网掩码是一个32位地址，用于屏蔽IP地址的一部分以区别网络标识和主机标识，并说明该IP地址是在局域网上，还是在远程网上。
  • 子网掩码不能单独存在，它必须结合IP地址一起使用。子网掩码只有一个作用，就是将某个IP地址划分成网络地址和主机地址两部分。
  • 子网掩码——屏蔽一个IP地址的网络部分的“全1”比特模式。对于A类地址来说，默认的子网掩码是255.0.0.0；对于B类地址来说默认的子网掩码是255.255.0.0；对于C类地址来说默认的子网掩码是255.255.255.0。

缓冲流

缓冲思想

  • 一次读写一个数组的速度明显比一次读写一个字节或字符的速度快很多，
  • 这是加入了数组这样的缓冲区效果，java本身在设计的时候，也考虑到了这样的设计思想(装饰设计模式)，所以提供了缓冲区流

缓冲原理

读

  • 缓冲流会先从文件将数据读取到缓冲区
  • 程序是从缓冲区中读取数据
  • 缓冲区数据读完后，然后重复过程

写

  • 程序会先将数据写入缓冲区中
  • 缓冲区写满后，缓冲流会将数据一次性写入文件，然后重复过程
    • 如果最后一次缓冲区没写满，需要
      • flush()
      • close()

  • 读取一个字节时

    • 缓冲区数据全部读取完毕，缓冲流再次一次性从文件中读取8192个

小结(安)

字节流

  • 读取一个字节，返回一个int
    • 会在字节前加24个0，这样一个字节就变成一个int
    • 为什么不返回一个字节
      • 因为读取的结束标记是返回-1
      • 如果-1是一个字节的-1，因为数据都是由字节组成的，难免数据中包含一个字节的-1，这样数据没读完就结束了。
      • 所以会在字节前加0 变成int 四个字节，这样表示一个字节的-1变成四个字节，结果肯定不是-1.
    • 读取中文时可能乱码
  • 写出一个字节 写出的是int 或者字节数组
    • 会将int的前24个0去除，这样就变成一个字节了。
    • 可以写出中文，以字节数组写出。
  • 一次读写一个字节，速度太慢
    1. 自己创建缓冲数组
    2. 使用缓冲流
      • 字节缓冲流，内置了一个8192大小的数组
      • 读取时先将数据读到缓冲区中，再从缓冲区读取一个字节。
        • 为什么从缓存区中读取一个字节速度就比直接读一个字节快？
          • 因为将数据读取到缓冲区中时是在内存中操作的，速度很快。降低硬盘读写次数就会提高效率
      • 写出时先将数据写到缓冲区中，缓冲区写满后，自动写出。若没满，可以flush或者close来写出。
        • 同样的，将数据从缓冲区写出时是在内存中操作的。

字符流

  • 读取一个字符，返回一个int
    • 在字符前加16个0
  • 写出一个字符 写出int string 或者字符数组
    • 写出int时，去除前16个0
  • 字符流不能操作非纯文本文件。

转换流

  • 字节流什么文件都能读写，而现在有纯文本文件每次都要将字节转为字符，不太方便，且有时会有编码问题。所以需要字符流。于是就出现了能将字节流转换位字符流的转换流。

    • 即如果文本文件编码与平台一致，那么直接用FileReader/FileWriter即可，如果不一致，我们自己用指定的码表将字节流转为字符流，

概述

进程 & 线程

进程

  • 正在执行中的程序，一个进程至少有一个线程。

线程

  • 进程中的一个负责程序执行的控制单元（执行路径）
  • 每一个线程都有自己运行的内容，可以称之为线程任务。

并行 & 并发

  • 并行指两个任务同时执行，(需要多核cpu)
  • 并发指两个任务都请求执行，同一时间只能执行一个任务，(cpu在不同任务之间快速切换)。

java程序运行原理

  • Java命令启动java虚拟机。启动jvm，等于启动了一个应用程序，也就是启动了一个进程，该进程会自动启动一个“主线程”，然后主线程去调用某个main方法。
  • jvm的启动是多线程的
    • jvm启动至少启动了主线程和垃圾回收线程。

线程状态

新建

  • 创建线程对象

就绪

  • 线程启动，有执行资格但没有获取cpu执行权
  • 新建→就绪
    • start()
  • 运行→就绪
    • 运行状态失去了执行权就变成就绪状态
  • 阻塞→就绪
    • 阻塞醒了以后得到执行资格就变成就绪状态

运行

  • 有执行资格且获取cpu执行权
  • 就绪→运行
    • 就绪状态抢到了执行权就变成运行状态

阻塞

  • 没有执行资格也没有cpu执行权
  • 运行→阻塞
    • 运行状态wait或者sleep就变成阻塞状态

死亡

  • 代码完毕线程死亡
    • run()结束
    • stop()(已过时)

创建线程

  1. 继承Thread

线程安全问题

线程安全问题产生的原因

  • 多个线程操作共享数据
  • 操作共享数据的线程代码有多条
  • 安
    • 比如卖票，现在标记是1，线程1判断大于0，还没减减，另一条线程也过来判断，这时也是大于0的，就出问题了。

解决

  • 思路就是一个线程，一组代码没执行完，不让别的线程执行。
  • 这就需要线程锁。没有锁进不来。

同步

什么情况下需要同步

  • 当多线程并发, 有多段代码同时执行时, 我们希望某一段代码执行的过程中CPU不要切换到其他线程工作. 这时就需要同步.
  • 如果两段代码是同步的, 那么同一时间只能执行一段, 在一段代码没执行结束之前, 不会执行另外一段代码.

同步代码块

  • 使用synchronized关键字加上一个锁对象来定义一段代码, 这就叫同步代码块
  • 多个同步代码块如果使用相同的锁对象, 那么他们就是同步的

同步函数

  • 返回值前加synchronized关键字

  • 非静态同步函数的锁是this

  • 静态的同步函数的锁是字节码对象

释放锁

  1. 执行完同步代码块
    • 我的理解是比如一个无限循环中，同步的代码是循环中的一部分，执行完这一部分也会释放锁。
  2. 在执行同步代码块的过程中，遇到异常而导致线程终止。
  3. 在执行同步代码块的过程中，执行了锁所属对象的wait()方法，这个线程会释放锁，进行对象的等待池

死锁

  • 多线程同步的时候, 如果同步代码嵌套, 使用相同锁, 就有可能出现死锁

  • 怎么通信
    • 如果希望线程等待, 就调用wait()
    • 如果希望唤醒等待的线程, 就调用notify();
    • 这两个方法必须在同步代码中执行, 并且使用同步锁对象来调用
  • 多个线程通信的问题

  • 锁

迭代器原理

  • 迭代器是对集合进行遍历,而每一个集合内部的存储结构都是不同的,所以每一个集合存和取都是不一样,那么就需要在每一个类中定义hasNext()和next()方法,这样做是可以的,但是会让整个集合体系过于臃肿,迭代器是将这样的方法向上抽取出接口,然后在每个类的内部,定义自己迭代方式。
  • 好处
    1. 规定了整个集合体系的遍历方式都是hasNext()和next()方法,
    2. 代码有底层内部实现,使用者不用管怎么实现的,会用即可

    • 查找Itr这个内部类,发现重写了Iterator中的所有抽象方法

xd-正则表达式

发表于 2018-11-22 | 更新于 2018-11-23 | 分类于 Java

概述

Pattern

  • java.util.regex.Pattern
    • 正则表达式的编译表示形式（正则对象）
    • 指定为字符串的正则表达式必须首先被编译为此类的实例
  • static Pattern compile(String regex)
    • 将给定的正则表达式编译到模式中
    • 即根据正则表达式获取正则对象
  • Matcher matcher(CharSequence input)
    • 根据正则对象获取匹配器对象
    • CharSequence
      • 是个接口，实现有String,StringBuffer....

Matcher

  • java.util.regex.Matcher
    • 匹配器对象
  • boolean matches()
    • 匹配整个字符串
    • 当且仅当整个区域序列匹配此匹配器的模式时才返回 true
  • boolean lookingAt()
    • 从字符串开头开始匹配，与matches()类似，但不需要匹配整个字符串
    • 当且仅当输入序列的前缀匹配此匹配器的模式时才返回 true
  • boolean find()
    • 查找是否有下一个匹配的序列
  • String group()
    • 获取下一个匹配的序列
    • 获取前要先查找
  • String replaceAll(String replacement)
    • 替换所有匹配序列

使用

特殊字符 （需要用\转义）

  • .
  • \
  • ^
  • $

预定义字符类

  • .
    • 任何字符。
  • \d

  • \D

    • 单词字符：[a-zA-Z_0-9]
    • 字母数字下划线
  • \W
    • 非单词字符

字符类

  • [abc]
    • a、b 或 c
  • [^abc]
    • 任何字符，除了 a、b 或 c（否定）
  • [a-zA-Z]
    • [a-z[A-Z]]
    • [a-z][A-Z]
    • 并集
    • a到 z 或 A到 Z，两头的字母包括在内（范围）
  • [0-9&&[^3]
    • 交集
    • 0-9所有数字除了3

    • 0或一个
  • *
    • 0或多个
  • +
    • 一个或多个
  • {n}

  • {n,}
    • 至少n个
  • {n,m}
    • n到m个

组

  • ()
    • 从左到右从1开始编号。
      • 以 ( 顺序分组

使用

  • \\组编号
    • 直接写组的编号即可
    • 注意转义,因为数字是字符
  • $组编号
    • 代表的是组的内容
    • 例如abccccdefgggghi将叠词用它自己替换即变成abcdefghi
      • str.replaceAll("(.)\\1+", "$1")

  • ^
    • 行的开头
  • $
    • 行的结尾
  • \b
    • 单词边界
  • \B
    • 非单词边界
  • 边界有什么用？（安）
    • 比如查找三个单词的匹配项，如果有四个五个。。的也会匹配
      • abc ades aaaaa ... 匹配的结果是abc ade aaa
      • 此时加上边界就可以只找三个的

    • 根据正则表达式用给定的字符串替换原字符串匹配的序列
    • str.replaceAll(regex, replacement)相当于
