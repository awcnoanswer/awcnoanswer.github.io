xd-网络编程

发表于 2018-11-23 | 更新于 2019-03-08 | 分类于 JavaSE

概述

  • 就是用来实现网络互连的不同计算机上运行的程序间可以进行数据交换。

三要素

IP

  • 每个设备在网络中的唯一标识
  • 每台网络终端在网络中都有一个独立的地址，我们在网络中传输数据就是使用这个地址。
  • ipconfig
    • 查看本机
  • ping
    • 测试连接
  • 127.0.0.1
    • 本地回路地址
  • localhost
    • 主机名
  • 255.255.255.255
    • 广播地址
  • IPv4
    • 4个字节组成，4个0-255。大概42亿，30亿都在北美，亚洲4亿。2011年初已经用尽。
  • IPv6
    • 8组，每组4个16进制数。

端口

  • 每个程序在设备上的唯一标识
  • 每个网络程序都需要绑定一个端口号，传输数据的时候除了确定发到哪台机器上，还要明确发到哪个程序。
  • 端口号范围从0-65535
  • 编写网络应用就需要绑定一个端口号，尽量使用1024以上的，1024以下的基本上都被系统程序占用了。
  • 常用端口
    • mysql: 3306
    • oracle: 1521
    • web: 80
    • tomcat: 8080
    • QQ: 4000

特点

  • 面向无连接，数据不安全，速度快。不区分客户端与服务端。

DatagramPacket

  • java.net.DatagramPacket
  • 数据包
构造
  • DatagramPacket(byte[] buf, int length)
    • 构造 DatagramPacket，用来接收长度为 length 的数据包
  • DatagramPacket(byte[] buf, int length, InetAddress address, int port)
    • 构造数据报包，用来将长度为 length 的包发送到指定主机上的指定端口号
方法
  • InetAddress getAddress()
    • 返回要发送到或从之接收的IP地址
  • int getPort()
    • 返回要发送到或从之接收的端口
  • byte[] getData()
    • 返回数据缓冲区（包含空的区域）
  • int getLength()
    • 返回将要发送或接收到的数据的长度

应用

发送
  • 创建DatagramSocket, 随机端口号,也可指定.
    • DatagramSocket sendSocket = new DatagramSocket([8888])
  • 创建DatagramPacket, 指定数据, 长度, 地址, 端口
    • DatagramPacket packet=new DatagramPacket(bytes,bytes.length, InetAddress.getByName("192.168.1.110"),9999)
  • 使用DatagramSocket发送DatagramPacket
    • sendSocket.send(packet)
  • 关闭DatagramSocket
    • sendSocket.close()
接收
  • 创建DatagramSocket, 指定端口号(与发送数据包端口一致)
    • DatagramSocket receiveSocket = new DatagramSocket(9999)
  • 创建DatagramPacket, 指定数组, 长度
    • DatagramPacket packet = new DatagramPacket(bytes,bytes.length)
  • 使用DatagramSocket接收DatagramPacket
    • receiveSocket.receive(packet)
  • 从DatagramPacket中获取数据
    • InetAddress address = packet.getAddress()
      • String ip=address.getHostAddress()
        • 获取ip
    • int port = packet.getPort()
      • 获取发送端口
    • String s = new String(packet.getData(), 0, packet.getLength())
      • 获取发送数据
  • 关闭DatagramSocket
    • receiveSocket.close()

应用

客户端
  • 创建Socket连接服务端(指定ip地址,端口号)通过ip地址找对应的服务器
    • Socket socket = new Socket("127.0.0.1", 9999)
  • 调用Socket的getInputStream()和getOutputStream()方法获取和服务端相连的IO流
    • InputStream in = socket.getInputStream()
      • 输入流可以读取服务端输出流写出的数据
    • OutputStream os = socket.getOutputStream()
      • 输出流可以写出数据到服务端的输入流
服务端
  • 创建ServerSocket(需要指定端口号)
    • ServerSocket server = new ServerSocket(9999)
  • 调用ServerSocket的accept()方法接收一个客户端请求，得到一个Socket
    • Socket socket = server.accept()
  • 调用Socket的getInputStream()和getOutputStream()方法获取和客户端相连的IO流
    • InputStream in = socket.getInputStream()
      • 输入流可以读取客户端输出流写出的数据
    • OutputStream os = socket.getOutputStream()
      • 输出流可以写出数据到客户端的输入流
服务端要接收多个客户端连接，需要用多线程
  • 将每个连接上的客户端要做的事封装成任务。
  • accept()是阻塞方法。
注意
  • 如果服务端和客户端都等待
    1. 数据没有刷新
      • 当使用BufferedReader包装流后，因为readLine()的结束标记是\r\n
        • 如果使用PrintStream的println()那么就相当于换行刷新
        • 如果使用PrintWriter的println()需要手动刷新flush()或者在构造时自动刷新设为true
        • 如果使用BufferedWriter那么需要手动newLine()
    2. 阻塞式方法没有结束标记
  • Socket的结束标记

    • void shutdownInput()

Http协议

概述

  • HyperText Transfer Protocle(超文本传输协议)
  • 用于定义web浏览器与服务器之间交换数据的过程及数据本身的格式
  • TCP协议对应于传输层，而HTTP协议对应于应用层，从本质上来说，二者没有可比性。Http协议是建立在TCP协议基础之上的。
    • 低层是tcp
  • FTP
    • 文件协议

特点

  1. 基于TCP/IP的高级协议
  2. 默认端口号:80
    • 即访问时http://www.baidu.com:80
      • 端口省略了
  3. 基于请求/响应模型的:一次请求对应一次响应
  4. 无状态的：每次请求之间相互独立，不能交互数据

历史版本

  • 1.0
    • 每一次请求响应都会建立新的连接
  • 1.1
    • 复用连接

子网掩码

概述

  • 随着互联网的发展，越来越多的网络产生，有的网络多则几百台主机，少则区区几台，这样就浪费了大量的IP地址，所以要划分子网。为了有序地区分和管理不同的子网，就需要使用子网掩码。
  • 子网掩码是一个32位地址，用于屏蔽IP地址的一部分以区别网络标识和主机标识，并说明该IP地址是在局域网上，还是在远程网上。
  • 子网掩码不能单独存在，它必须结合IP地址一起使用。子网掩码只有一个作用，就是将某个IP地址划分成网络地址和主机地址两部分。
  • 子网掩码——屏蔽一个IP地址的网络部分的“全1”比特模式。对于A类地址来说，默认的子网掩码是255.0.0.0；对于B类地址来说默认的子网掩码是255.255.0.0；对于C类地址来说默认的子网掩码是255.255.255.0。

xd-正则表达式

发表于 2018-11-22 | 更新于 2019-03-08 | 分类于 JavaSE

概述

Pattern

  • java.util.regex.Pattern
    • 正则表达式的编译表示形式（正则对象）
    • 指定为字符串的正则表达式必须首先被编译为此类的实例
  • static Pattern compile(String regex)
    • 将给定的正则表达式编译到模式中
    • 即根据正则表达式获取正则对象
  • Matcher matcher(CharSequence input)
    • 根据正则对象获取匹配器对象
    • CharSequence
      • 是个接口，实现有String,StringBuffer....

Matcher

  • java.util.regex.Matcher
    • 匹配器对象
  • boolean matches()
    • 匹配整个字符串
    • 当且仅当整个区域序列匹配此匹配器的模式时才返回 true
  • boolean lookingAt()
    • 从字符串开头开始匹配，与matches()类似，但不需要匹配整个字符串
    • 当且仅当输入序列的前缀匹配此匹配器的模式时才返回 true
  • boolean find()
    • 查找是否有下一个匹配的序列
  • String group()
    • 获取下一个匹配的序列
    • 获取前要先查找
  • String replaceAll(String replacement)
    • 替换所有匹配序列

使用

特殊字符 （需要用\转义）

  • .
  • \
  • ^
  • $

预定义字符类

  • .
    • 任何字符。
  • \d
    • 数字：[0-9]
  • \D
    • 非数字
  • \s
    • 空白字符:[ \t\n\x0B\f\r]
  • \S
    • 非空白字符
  • \w
    • 单词字符：[a-zA-Z_0-9]
    • 字母数字下划线
  • \W
    • 非单词字符

字符类

  • [abc]
    • a、b 或 c
  • [^abc]
    • 任何字符，除了 a、b 或 c（否定）
  • [a-zA-Z]
    • [a-z[A-Z]]
    • [a-z][A-Z]
    • 并集
    • a到 z 或 A到 Z，两头的字母包括在内（范围）
  • [0-9&&[^3]
    • 交集
    • 0-9所有数字除了3

数量

  • ?
    • 0或一个
  • *
    • 0或多个
  • +
    • 一个或多个
  • {n}
    • n个
  • {n,}
    • 至少n个
  • {n,m}
    • n到m个

组

分组

  • ()
    • 从左到右从1开始编号。
      • 以 ( 顺序分组
      • 组 0 始终代表整个表达式

使用

  • \\组编号
    • 直接写组的编号即可
    • 注意转义,因为数字是字符
  • $组编号
    • 代表的是组的内容
    • 例如abccccdefgggghi将叠词用它自己替换即变成abcdefghi
      • str.replaceAll("(.)\\1+", "$1")

边界

  • ^
    • 行的开头
  • $
    • 行的结尾
  • \b
    • 单词边界
  • \B
    • 非单词边界
  • 边界有什么用？（安）
    • 比如查找三个单词的匹配项，如果有四个五个。。的也会匹配
      • abc ades aaaaa ... 匹配的结果是abc ade aaa
      • 此时加上边界就可以只找三个的

String类中的正则

  • boolean matches(String regex)

    • 判断字符串是否匹配正则
    • boolean b = str.matches(regex)相当于

  • String[] split(String regex)

    • 根据正则表达式切割字符串为字符数组
  • String replaceAll(String regex, String replacement)
    • 根据正则表达式用给定的字符串替换原字符串匹配的序列
    • str.replaceAll(regex, replacement)相当于

特点

  • 自己调用自己
  • 确定结束条件

好处 & 弊端

  • 好处
    • 不用知道循环次数。
  • 弊端
    • 调用次数过多，栈内存会溢出。

例

  1. 求一个数的阶乘

  2. 求一个数的阶乘结果后边有多少个0

    • 规律
      • 只要一个数是5的倍数，那么它跟偶数相乘末尾肯定有个0，所以一个数有多少个5，结果末尾就有多少个0
      • 每隔5个的数就包含一个5。num/5可以求出包含一个5的数有多少个
        • 5*1 5*2 5*3 5*4 5*5 ...
        • 而这些数中每隔5个的数包含两个5，num/5/5可以求出包含两个5的数有多少个
          • 5*5*1 5*5*2 5*5*3 5*5*4 5*5*5 ...
          • 在这些数中每隔5个的数包含三个5，num/5/5/5可以求出包含三个5的数有多少个
          • ……
          • 一直到最后数不够5个了，就没有包含5的数了。

  3. 斐波那契数列

    • 问题
      • 一对刚出生的兔子，一个月后长成大兔子，此后每月都会生出一对兔子，问n月兔子的总对数。
    • 规律
      • 1 1 2 3 5 8 13 ...
      • 可以看出除了第一第二月，后边每月是前边两月的和。

类加载

概述

  • 当程序要使用某个类时，如果该类还未被加载到内存中，则系统会通过加载，连接，初始化三步来实现对这个类进行初始化。

加载

  • 就是指将class文件读入内存，并为之创建一个Class对象。任何类被使用时系统都会建立一个Class对象。

连接

  • 验证 是否有正确的内部结构，并和其他类协调一致
  • 准备 负责为类的静态成员分配内存，并设置默认初始化值
  • 解析 将类的二进制数据中的符号引用替换为直接引用

初始化

  • 实例对象初始化过程

时机

  • 创建类的实例
  • 访问类的静态变量，或者为静态变量赋值
  • 调用类的静态方法
  • 使用反射方式来强制创建某个类或接口对应的java.lang.Class对象
  • 初始化某个类的子类
  • 直接使用java.exe命令来运行某个主类

类加载器

概述

  • 负责将.class文件(可能在本地，也可能在内存)加载到内存中，并为之生成对应的Class对象。

分类&作用

Bootstrap ClassLoader

  • 根类加载器
  • 也被称为引导类加载器，负责Java核心类的加载
  • 比如System,String等。在JDK中JRE的lib目录下rt.jar文件中

Extension ClassLoader

  • 扩展类加载器
  • 负责JRE的扩展目录中jar包的加载。
  • 在JDK中JRE的lib目录下ext目录

Sysetm ClassLoader

  • 系统类加载器(AppClassLoader)
  • 负责在JVM启动时加载来自java命令的class文件，以及classpath环境变量所指定的jar包和类路径

父类委托机制

  • 保证字节码只被加载一次，避免类重复加载。
  • 要加载一个类
  • Sysetm ClassLoader首先让Extension ClassLoader找
  • Extension ClassLoader让Bootstrap ClassLoader找
  • Bootstrap ClassLoader找不到，Extension ClassLoader才找
  • Extension ClassLoader找不到，Sysetm ClassLoader才找
  • Sysetm ClassLoader找不到，ClassNotFoundException
  • 安
    • 总之就是一级一级找上边加载，最后找到最顶层，然后上边一级一级往下轮着加载，不管哪一级加载到就停止，一直到最底层如果还没加载到，就报错。
    • 从哪一层开始找，最后到那一层结束，所以class文件不能乱放

java执行过程

  1. .java原文件 编译
  2. .class字节码文件
  3. 类加载器把.class字节码文件装载到内存运行环境中，生成Class字节码对象。
  • 由上述过程可知，根据Class字节码对象可以知道是哪个类加载器加载的

概述

  • Lambda 是一个 匿名函数，我们可以把 Lambda表达式理解为是 一段可以传递的代码（将代码像数据一样进行传递）
  • Lambda需要函数式接口的支持

并行流/串行流

并行流

  • 把一个内容分成多个数据块，并用不同的线程分别处理每个数据块的流

Fork/Join框架

  • 采用 “工作窃取”模式（work-stealing）
    • 当执行新的任务时它可以将其拆分分成更小的任务执行，并将小任务加到线程队列中，然后再从一个随机线程的队列中偷一个并把它放在自己的队列中
  • 与一般线程池实现的比较
    • 在一般的线程池中,如果一个线程正在执行的任务由于某些原因无法继续运行,那么该线程会处于等待状态
    • 在fork/join框架实现中,如果某个子问题由于等待另外一个子问题的完成而无法继续运行.那么处理该子问题的线程会主动寻找其他尚未运行的子问题来执行

函数式接口

  • 接口中只有一个抽象方法，即为函数式接口
  • @FunctionalInterface
    • 标明是函数式接口

内置函数式接口

语法

  • 左右遇一括号省
    • 左侧参数为一省略左侧小括号
    • 右侧语句为一省略return和大括号
  • 左侧推断类型省

操作符->

  • 箭头操作符或Lambda操作符
  • 左侧
    • 指定了 Lambda 表达式需要的所有参数
    • (参数列表)
  • 右侧
    • 指定了 Lambda 体，即 Lambda 表达式要执行的功能
    • {lambda体}

格式

  1. () -> System.out.println("hello lambda")
    • 无参
  2. (args) -> System.out.println(args)
    • 一个参数
  3. args -> System.out.println(args)
    • 一个参数可以省略小括号()

  4. .

    • 两个参数，且又返回值
  5. (x,y) -> x+y;
    • lambda体只有一句时，return和{}可以省略

  6. .

    • 参数的类型可以省略，因为可由编译器推断得出，称为“类型推断”

类型推断

  • Lambda 表达式中无需指定类型，程序依然可以编译，这是因为 javac 根据程序的上下文，在后台推断出了参数的类型。
  • Lambda 表达式的类型依赖于上下文环境，是由编译器推断出来的。这就是所谓的“类型推断”

方法引用

  • 当要传递给Lambda体的操作，已经有实现的方法了，可以使用方法引用

前提

  • Lambda体方法引用的参数列表和返回值类型必须与函数式接口中抽象方法的参数和返回值保持一致

格式

  1. 对象::实例方法
  2. 类名::静态方法

Stream API

概述

  • java.util.stream .*
  • 是数据渠道，用于操作数据源（集合、数组等）所生成的元素序列
  1. Stream 自己不会存储元素
  2. Stream 不会改变源对象。相反，他们会返回一个持有结果的新Stream
  3. Stream 操作是延迟执行的。这意味着他们会等到需要结果的时候才执行
    • 即中间操作不会进行操作，终止操作时，一次性执行全部内容，即“惰性求值”

  • default Stream<E> parallelStream()

Optional

  • java.util.Optional 是一个容器类，代表一个值存在或不存在，
  • 原来用 null 表示一个值不存在，现在 Optional 可以更好的表达这个概念。并且可以避免空指针异常

API

  • static of(T t)
    • 创建Optional
  • static empty()
    • 创建一个空的Optional
  • static ofNullable(T t)
    • 若t不为null，创建Optional，否则创建空的Optional
  • isPresent()
    • 是否包含值
  • orElse(T t)
    • 如果调用对象包含值，返回该值，否则返回t
  • orElse(Supplier s)
    • 如果调用对象包含值，返回该值，否则返回s获得的值
  • map(Function f)
    • 如果有值对其处理，并返回处理后的Optional，否则返回Optional.empty()
  • flatMap(Function mapper)
    • 类似map，要求返回值必须为Optional
