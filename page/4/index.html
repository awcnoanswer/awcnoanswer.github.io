ECMAScript

概述

  • ECMAScript实际上是一种脚本在语法和语义上的标准。
  • ECMAScript是JavaScript的规格
  • JavaScript是ECMAScript的一种实现

实现

  • JavaScript
  • JScript
  • ActionScript

说明

  • 三个都是ECMAScript。但是在操作浏览器对象等方面又有各自独特的方法，这些都是各自语言的扩展。
  • 类似于ASCII码表，以后的码表都包含这个。这三个脚本语言都包含ECMAScript而又有各自的东西(安)

DHTML

  • 不是一门语言，是多项技术综合体的简称
  • html
    • 负责提供标签，进行数据的封装。便于对数据的操作。
    • 用标签封装数据。
  • css
    • 负责提供样式。
    • 对数据定义样式。
  • dom
    • 负责将标签文档解析，封装对象，便于操作。
    • 解析文档封装对象。
  • javascript
    • 负责提供程序设计语言，对页面的对象进行逻辑操作。
    • 页面的行为定义，就是页面的动态效果。
  • DHTML+XMLhttpRequest=AJAX

JavaScript

概述

  • 基于对象和事件驱动的解释性脚本语言。
    • 基于对象
      • JS本身就有一些现成的对象可供程序员使用，也可自定义对象。
    • 事件驱动
      • JS代码写好后，需要外界触发后，方可运行
      • 例如：单击事件，定时执行。。。
    • 解释性
      • 每次运行JS代码时，得需要将原代码一行一行的解释执行
      • 相对编译型语言（例如：Java、C++）执行速度相对较慢
    • 基于浏览器的动态交互网页技术
      • JS嵌入到HTML中，可以不需要服务器支持，直接由浏览器解释执行
      • JS嵌入到JSP或Servlet中，必需要服务器支持，直接由浏览器解释执行

特点

  • 交互性
  • 安全性
    • js不能访问本地磁盘
  • 跨平台

基于对象&面向对象(网)

  • 基于对象
    • 先有一个具体的对象，然后在这个对象的基础上创建新的对象
    • 一个工程师建了一栋房子，然后其它的工程师按照这个房子的样子去建造其它的房子
  • 面向对象
    • 先有一个抽象的对象描述，然后以此为蓝本构建具体对象
    • 一个工程师再图纸上设计出一栋房子的样子，然后其它工程师按照这个图纸的设计去建造房子

引入方式

  1. 内嵌脚本
    • <input type="button" value="button" onclick="alert('xxx')"/>
  2. 内部脚本
    • 将js代码封装到<script>标签中
    • <script type="text/javascript">js代码<script>
      • type可以省略 默认。
      • <script>标签可以出现在html页面的任意位置，但实际操作中，一般会写在<head>标签中。
      • 在不影响html功能的前提下越晚加载越好
  3. 外部脚本
    • 将js代码封装成js文件，使用时通过<script>标签的src属性导入
    • <script src="xx.js></script>"
      • 外部的js文件中不能包含<script>标签。
      • 在导入外部js文件的<script>标签中不能在写js代码(标签体为空)。

  • DOM level 3

解析

dom解析

  • w3c标准解析
  • 将标记型文档解析成DOM树，并将树中的内容都封装成节点对象。
    • DOM树
      • 按照标签的层次关系体现出标签的所属，形成的一个树状结构。
    • 节点(Node)
      • DOM树中的标签，属性及文本。
      • 节点也成为对象。
      • 标签通常也称为页面中的元素。
  • 好处
    • 可以对节点进行任意操作。
  • 弊端
    • 需要将整个标记型文档加载进内存，如果文档过大，浪费内存空间。

SAX解析

  • 民间方式。
  • 基于事件驱动的解析。
  • 依序读入文件并产生相应事件，可以处理任意大小的文件。
  • 只能读取，不能修改。不能往回读。

  • 安
    • 如有三个div块

    • fixed
      • 生成绝对定位的元素，相对于浏览器窗口进行定位。元素的位置通过 “left”, “top”, “right” 以及 “bottom” 属性进行规定。

例

  2. 求一个数的阶乘结果后边有多少个0

    • 规律
      • 只要一个数是5的倍数，那么它跟偶数相乘末尾肯定有个0，所以一个数有多少个5，结果末尾就有多少个0
      • 每隔5个的数就包含一个5。num/5可以求出包含一个5的数有多少个
        • 5*1 5*2 5*3 5*4 5*5 ...
        • 而这些数中每隔5个的数包含两个5，num/5/5可以求出包含两个5的数有多少个
          • 5*5*1 5*5*2 5*5*3 5*5*4 5*5*5 ...
          • 在这些数中每隔5个的数包含三个5，num/5/5/5可以求出包含三个5的数有多少个
          • ……
          • 一直到最后数不够5个了，就没有包含5的数了。

  3. 斐波那契数列

    • 问题
      • 一对刚出生的兔子，一个月后长成大兔子，此后每月都会生出一对兔子，问n月兔子的总对数。
    • 规律
      • 1 1 2 3 5 8 13 ...
      • 可以看出除了第一第二月，后边每月是前边两月的和。

  4. 约瑟夫环（非递归，集合）

    • 问题
      • n个人围成一个环，从头开始数数，数到3的倍数就把这个位置的人杀掉，那么n个人中哪个数是活到最后的。

发表于 2018-11-23 | 更新于 2018-12-23 | 分类于 Java

Junit4

注解

  • @Test
    • public void method()
    • 测试注释
    • 指示方法可以作为一个测试用例
  • @Before
    • public void method()
    • 该方法必须在类中的每个测试之前执行，以便执行测试某些必要的先决条件。
    • 会在每个用例运行之前都运行一次
  • @BeforeClass
    • public static void method()
    • 在测试类里所有用例运行之前，运行一次这个方法
    • 只会运行一次，一般用于创建数据库连接、读取文件等
  • @After
    • public void method()
    • 该方法在执行每项测试后执行
    • 如执行每一个测试后重置某些变量，删除临时变量等
  • @AfterClass
    • public static void method()
    • 测试类里所有用例运行之后，运行一次。用于处理一些测试后续工作
    • 如清理数据，从数据库断开连接
  • @Ignore
    • public static void method()
      • 当想暂时禁用特定的测试执行可以使用忽略注释
      • 每个被注解为@Ignore的方法将不被执行。

断言

  • void assertEquals([String message], expected value, actual value)
    • 断言两个值相等
    • 值可能是类型有 int, short, long, byte, char or java.lang.Object.
    • 第一个参数是一个可选的字符串消息
  • void assertTrue([String message], boolean condition)
    • 断言一个条件为真
  • void assertFalse([String message],boolean condition)
    • 断言一个条件为假
  • void assertNotNull([String message], java.lang.Object object)
    • 断言一个对象不为空(null)
  • void assertNull([String message], java.lang.Object object)
    • 断言一个对象为空(null)
  • void assertSame([String message], java.lang.Object expected, java.lang.Object actual)
    • 断言两个对象引用相同的对象
  • void assertNotSame([String message], java.lang.Object unexpected, java.lang.Object actual)
    • 断言两个对象不是引用同一个对象
  • void assertArrayEquals([String message], expectedArray, resultArray)
    • 断言预期数组和结果数组相等
    • 数组的类型可能是 int, long, short, char, byte or java.lang.Object.

简单工厂（静态工厂）

概述

优缺点

优点

  • 客户端不需要负责对象的创建，从而明确了各个类的职责

缺点

  • 静态工厂类负责所有对象的创建，如果有新的对象增加，或者某些对象的创建方式不同，就要不断的修改工厂类，不利于后期维护。

工厂方法（一般工厂）

概述

  • 抽象工厂类负责定义创建对象的接口，具体对象的创建由继承抽象工厂的具体类实现。
  • 其实就是创建一个抽象的顶级接口，然后具体对象有一个针对的工厂类实现顶级接口。新增对象时，新增一个对象工厂即可，不必修改其它工厂。

适配器

概述

  • 假如有个接口有许多方法，我们要实现接口，就要覆写所有的方法，有些方法我们根本用不到。
  • 现在有个类或抽象类，实现接口，把所有方法或不常用的方法覆写。可能方法内部什么也没做，就是简单的覆写。
  • 这样我们继承这个类或抽象类。只需要覆写想要覆写的方法或者实现没有覆写的抽象方法即可。

例

  • Servlet接口 GenericServlet抽象类
  • Servlet接口中有五个方法，如果我们实现了Servlet接口，就必须要覆写这五个方法，有的方法，我们根本用不到。
  • 现在有了GenericServlet抽象类实现了Servlet接口，我们继承GenericServlet即可。

例

    • MouseListener中有多个方法
      • MouseAdapter全部实现，方法体为空，使用时只继承适配器类即可，用哪个方法复写哪个。

代理模式

  • Proxy Pattern

结构

  • 目标对象
    • 真正要访问的对象
  • 代理对象
  • 代理类
    • 用来生成代理对象
  • 目标对象和代理对象实现同一接口

原理(安)

  • 目标对象和代理对象有着相同的功能。
  • 做一件事，内部其实还是目标对象在做，只是表面是代理对象在执行行为。
  • 代理对象的行为内部在目标对象做的事前后还可以自己做一些事情。
  • 例
    • 潘金莲和王婆有相同功能（实现同一个接口）抛媚眼
    • 潘金莲（目标对象）给西门庆抛媚眼（事情），不直接这么干，表面上由王婆（代理对象）抛媚眼，其实真正抛媚眼的还是潘金莲。
    • 在王婆的抛媚眼方法内部，执行的是潘金莲的抛媚眼，而在潘金莲抛媚眼前后，王婆还可以做一些事情(如之前拿些好处，之后干掉武大郎)

动态代理

概述

  • 动态代理可以直接给某一个目标对象生成一个代理对象，而不需要代理类存在。
  • 目标对象和代理对象实现同一接口是为了保证方法一致
  • 动态代理与代理模式原理是一样的，只是它没有具体的代理类，直接通过反射生成了一个代理对象。

jdk动态代理

概述
  • 对装饰者设计模式简化。使用前提：必须有接口
  • 接口+实现类
Proxy
  • 返回代理对象的名称
    • $Proxy数字
方法
  • static Object newProxyInstance(ClassLoader loader,Class<?>[] interfaces,InvocationHandler h)
    • 返回一个指定接口的代理类实例。
    • ClassLoader
      • 类加载器。固定写法，和被代理类使用相同的类加载器即可,动态代理类 运行时创建，任何类都需要类加载器将其加载到内存。
        • 当前类.class.getClassLoader()
        • 代理类实例.getClass().getClassLoader()
    • Class[] interface
      • 代理类要实现的接口。固定写法，和被代理类使用相同的接口即可
        • 代理类实例.getClass().getInterfaces()
        • new Class[]{被代理类接口.class}
    • InvocationHandler
      • 接口，一般采用匿名内部类
      • 代理类的每一个方法执行时，都将调用一次invoke()
      • Object invoke(Object proxy,Method method,Object[] args)
        • proxy
          • 代理对象,一般用不着
        • method
          • 代理对象当前执行的方法的描述对象（反射）
          • 获取方法名
            • method.getName()
          • 执行方法
            • method.invoke(实现类对象，实际参数)
        • args
          • 方法实际参数
        • 返回值
          • 即目标方法的返回值
  • 安
    • 我们要执行一个目标类A的目标方法add()
      • 原
        • A a=new A()
        • a.add()
      • 使用代理
        • A $a=Proxy.newProxyInstance(classLoader,interfaces,h)
          • 获取A的代理对象$a
        • $a.add()
          • 代理类$a执行方法add()时，会自动调用h的invoke()方法
        • invoke()方法中的method.invoke(a，实际参数)
          • 执行的是目标类的方法。即a.add()
          • 这样就通过代理完成了目标类A的目标方法add()
        • 而在invoke()方法中调用method.invoke(a，实际参数)前后还可以做一些其它事情。
          • 这样通过代理我们不仅可以实现原来调用目标方法的目的，还可以在目标方法执行前后做一些事。
